# 量化选股项目任务表

更新日期：2026-04-23

## 一、项目目标

把当前项目从“能跑的简单选股脚本”逐步升级为“稳定的数据平台 + 可扩展的策略研究系统 + 后续可接 AI 评分与交易执行”的完整量化项目。

当前原则：

- 先把数据层和工程结构打稳
- 先做可复用、可验证、可迁移的方案
- AI 放在规则筛选之后，不一开始就堆复杂度
- 实盘交易优先研究正式接口，不用 UI 自动点击做核心方案

## 二、阶段任务

### 阶段 1：稳定数据获取

目标：把日线数据获取做成稳定、可恢复、可追踪。

任务：

1. 以 TUShare 作为主数据源
2. AKShare 保留为备用数据源，不作为默认主源
3. 完成 token 本地文件读取，不提交到 GitHub
4. 增加请求限频控制
5. 增加失败重试机制
6. 增加失败股票清单与补抓机制
7. 增加抓取日志与成功率统计
8. 明确股票列表、日线、分钟线、候选结果的目录结构

完成标准：

- 全市场日线抓取过程中，失败股票可追踪
- 可重复运行且支持增量更新
- 不因单只股票失败导致整批任务中断

### 阶段 2：升级数据存储架构

目标：从 CSV 缓存过渡到更适合研究和扩展的数据层。

建议方案：

- 原始数据：Parquet
- 查询分析：DuckDB
- 任务状态/轻量配置：SQLite 或 JSON

任务：

1. 设计 `data/raw`、`data/feature`、`data/result` 分层
2. 日线数据从 CSV 迁移到 Parquet
3. 建立 DuckDB 数据目录与查询入口
4. 保留 CSV 导出，仅作为查看和分享用途
5. 统一数据字段命名规范
6. 明确复权、频率、时区、交易日字段标准

完成标准：

- 可以直接按股票、日期、板块快速查询
- 后续 OpenBB、pandas、DuckDB 可以直接接入

### 阶段 3：重构策略研究层

目标：把“脚本式选股”升级成“可组合策略框架”。

任务：

1. 拆分数据获取、特征计算、选股逻辑、回测逻辑
2. 建立统一特征层
3. 把 KDJ、MACD、跌幅等指标沉淀成标准特征
4. 增加行业、板块、成交额、波动率等辅助特征
5. 建立策略参数配置文件
6. 增加策略结果版本化保存

完成标准：

- 一个策略可以复用同一套特征数据
- 同时支持多个策略并行研究

### 阶段 4：增强回测与评估

目标：让策略效果可量化、可复盘、可比较。

任务：

1. 完善收益率、胜率、回撤、盈亏比等指标
2. 增加基准对比
3. 增加参数敏感性测试
4. 增加样本外测试
5. 增加组合层面的回测，而不只是单股信号统计
6. 输出结构化回测报告

完成标准：

- 能比较多个策略、多个参数组
- 能区分训练期与验证期

### 阶段 5：可视化与研究面板

目标：建立面向研究和决策的可视化界面。

任务：

1. 引入 OpenBB 或自建面板进行数据展示
2. 展示日线、分钟线、成交量、指标、候选股列表
3. 展示回测表现、信号分布、行业分布
4. 支持按日期回看候选结果
5. 支持导出图片、报表和候选清单

完成标准：

- 可以快速查看某天为何选出某只股票
- 可以快速对比板块和策略表现

### 阶段 6：AI 评分与非结构化信息融合

目标：在量化初筛后，用 AI 做二次复评。

任务：

1. 建立“规则初筛 -> AI 复评”的两阶段流程
2. AI 输入包括：
   - 候选股票基础特征
   - K线图/分时图
   - 新闻摘要
   - 公告和赛道信息
3. 增加新闻抓取与文本清洗
4. 增加新闻情绪和主题标签
5. 增加 AI 评分记录和解释结果

完成标准：

- AI 不直接替代量化筛选
- AI 输出可追踪、可复盘、可比较

### 阶段 7：分钟级数据与交易执行

目标：从日线研究逐步扩展到更高频的数据和执行层。

任务：

1. 增加分钟线数据获取与存储
2. 研究分钟级策略和风险控制
3. 建立仿真交易流程
4. 优先调研正式券商接口或量化终端
5. 建立下单、撤单、持仓、风控、日志模块
6. 严禁把 UI 自动点击作为核心实盘方案

完成标准：

- 先仿真稳定，再考虑实盘
- 没有正式接口时，只做研究和半自动提醒

## 三、当前建议的技术路线

短期：

- TUShare 主数据源
- CSV 继续可用，但只作为过渡层
- 先补齐重试、限频、失败清单

中期：

- Parquet + DuckDB 作为核心数据层
- OpenBB / 自建可视化面板
- 多策略与回测框架

长期：

- AI 评分、新闻挖掘、板块分析
- 分钟级策略
- 仿真交易与正式接口执行

## 四、近期优先级

建议按下面顺序推进：

1. 完善 TUShare 的重试、限频、失败补抓
2. 设计并迁移到 Parquet + DuckDB
3. 重构特征层和策略层
4. 完善回测报告
5. 接入可视化面板
6. 增加 AI 评分和新闻挖掘
7. 最后研究自动交易

## 五、备注

- 本项目当前仍以“研究和选股”为主，不应过早耦合交易执行
- 任何涉及实盘自动下单的能力，都必须在正式接口、风控和审计能力成熟后再推进
- 后续每完成一个阶段，建议更新本任务表
